# A4 赛题需求摘要

## 赛题背景

A4 题要求基于翼辉开放云边端协同系统完成智能仓储仿真。项目重点不是传统后台管理系统，而是在云边端平台内完成任务分配、AGV 调度、货架机器人上架和一致性校验。

## 关键对象

- 传送带和拣货缓冲区。
- 多台 AGV。
- 交接区。
- 货架机器人。
- 货架、盘位和库存映射。
- 云侧调度、边侧路径规划、端侧执行。

## 三个功能需求

1. 多 AGV 在网格仓库中完成取货、运输、交接，不能发生顶点碰撞或对向换边冲突。
2. 货架机器人按同体积批次上架，批次总体积不超过 1.0。
3. 交接区容量、盘位占用、库存映射必须始终一致。

## 评分风险

- AGV 碰撞或换边冲突。
- 交接区违规堆放。
- AGV 或货架机器人超容量。
- 货物写入非法或已占用盘位。
- 库存映射与真实盘位状态不一致。
- 过度依赖外部 WMS 导致平台内不能独立运行。

## 本项目策略

先用标准库实现可测试内核，再通过 `platform_modules/` 接入翼辉平台。WMS 作为展示和扩展，不参与核心得分链路。
