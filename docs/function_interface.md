# 功能接口草案

## AGV 状态接口

- 输入：AGV 编号、当前位置、载货列表、可用状态、当前路径。
- 输出：路径指令、等待指令、停止指令。
- V1 状态机：`IDLE`、`TO_PICKUP`、`PICKING`、`TO_HANDOFF`、`WAIT_HANDOFF`、`DROPPING`、`RETURNING`、`BLOCKED`、`ERROR`。
- 状态转移必须记录日志，并在取货、交接和返航前检查容量、路径预约和资源锁。
- 待确认：平台坐标系、时间 tick、指令下发格式、异常回调。

## 传送带/缓冲区接口

- 输入：传送带编号、位置、队列货物、占用状态。
- 输出：取货任务确认。
- 待确认：队列读取方式、缓冲区占用时长、货物唯一编号。

## 交接区接口

- 输入：交接区编号、目标货架、当前等待货物、容量。
- 输出：是否可放入、释放事件。
- V1 使用 `handoff_zone_lock:<zone_id>` 保护同一时间窗的交接操作；容量不足时 AGV 进入 `WAIT_HANDOFF`。
- 待确认：容量单位、占用释放事件来源。

## 货架机器人接口

- 输入：机器人编号、可服务货架、忙闲状态、可处理盘位。
- 输出：上架批次、目标盘位。
- 待确认：真实处理耗时、失败重试、机器人状态回调。

## 盘位/库存映射接口

- 输入：货架编号、盘位编号、盘位容量、占用货物。
- 输出：`cargo_id -> shelf_id, slot_id`。
- 待确认：平台是否已有库存表、盘位是否支持体积约束、提交格式。

## 平台待确认项

- 最终评测是否只在翼辉平台内运行。
- 是否允许外部服务。
- 是否允许第三方 Python 库。
- 地图拓扑读取方式。
- 平台是否提供缓冲区、交接区、窄路等资源锁原语；如果没有，需要在边侧调度模块内部维护。
- 项目导出和提交方式。
