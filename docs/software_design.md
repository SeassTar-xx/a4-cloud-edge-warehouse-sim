# 软件设计

## 云边端架构

- 云侧：全局任务分配、策略参数、整体状态汇总。
- 边侧：实时路径规划、路权预约、局部重规划、冲突避免。
- 端侧：AGV 和货架机器人执行具体动作，并回传状态。

## 核心模块

- 调度器：选择 AGV、传送带、货物批次和目标交接区。
- 路径规划器：时间扩展 A*，动作包含上下左右和等待。
- 预约表：记录格点和边在时间维度上的占用。
- 货架机器人策略：同体积、容量 1.0 以内批处理。
- 约束检查器：返回可读错误列表。

## 数据流

传送带队列进入云侧调度器，调度器选择任务后交给边侧路径规划器，路径结果下发给端侧 AGV。AGV 到达交接区后，货架机器人根据交接区等待货物选择批次并写入盘位和库存映射。

## 状态机

AGV 状态：空闲 -> 去取货 -> 装载 -> 去交接区 -> 卸货 -> 空闲。  
货架机器人状态：空闲 -> 选择批次 -> 上架 -> 更新库存 -> 空闲。  
交接区状态：空闲 -> 占用 -> 等待处理 -> 释放。

## 算法设计

调度采用保守评分：距离越近越优，交接区越拥堵惩罚越高，机器人忙碌增加惩罚，同目标合并给收益。路径规划采用时间扩展 A*，在搜索中检查顶点预约和边预约，避免同格冲突与对向换边。

## V1 CE-SafeScheduler

V1 目标是保命稳定，不追求最短时间。核心策略如下：

- 预约表同时管理顶点、边和资源锁，资源锁包括 `buffer_lock`、`handoff_zone_lock` 和 `narrow_road_lock`。
- `reserve_path` 是原子操作；如果中途出现顶点、边或资源冲突，会回滚已经写入的预约。
- 路径规划使用时间扩展 A*，状态为 `(point, time)`，动作包含上下左右和 WAIT；找不到路径时返回明确失败原因。
- 调度器只给 `IDLE` AGV 派任务；如果交接区容量不足、AGV 容量不足或没有安全路径，则不派发任务。
- AGV 使用状态机逐 tick 推进，状态包括 `IDLE`、`TO_PICKUP`、`PICKING`、`TO_HANDOFF`、`WAIT_HANDOFF`、`DROPPING`、`RETURNING`、`BLOCKED`、`ERROR`。
- 只要交接区、资源锁或路径存在不确定冲突，AGV 选择 WAIT 或进入 BLOCKED 等待后续重规划。
- 货架机器人只处理自己负责的货架，每次只处理同体积货物，批次总体积不超过 1.0，上架后立即更新库存映射。

## WMS 扩展定位

复制的 WMS 位于 `wms_extension/`，用于云侧仓储业务展示、库存映射展示和演示扩展。评测核心必须由 `a4_core/` 独立完成。
